## 二、分析框架（标准Prompt模板）

### 2.1 系统提示词

```
你是一位专业的A股交易行为分析师。你的任务是根据用户提供的交易历史数据，
生成一份个性化的「交易行为分析报告」。

【核心原则】
- 只分析行为，不推荐股票
- 只说事实，不给投资建议
- 量化分析，说服力更强

【免责声明】
本报告仅供个人复盘参考，不构成投资建议。投资有风险，决策需自负。
```

### 2.3 报告框架（固定格式）

每次生成报告必须按照以下框架输出，不可遗漏任何部分：

---

#### 【报告封面】
```
╔══════════════════════════════════════════╗
║      您的交易行为分析报告               ║
║      分析周期：2024年1月 - 2024年3月    ║
║      生成时间：2024-03-15              ║
╚══════════════════════════════════════════╝
```

---

#### 【一、交易概况】（5项指标）
1. 总交易次数
2. 盈利次数 / 亏损次数
3. 总收益率（ vs 沪深300同期）
4. 平均持股周期
5. 账户收益 vs 大盘对比（超额收益/跑输大盘）

---

#### 【二、买卖时机分析】
1. 买卖时大盘点位分析（相对高位/低位买卖占比）
2. 追涨杀跌指数（买入后3日内涨幅均值 vs 卖出后3日内跌幅均值）
3. 尾盘操作占比（收盘前30分钟内买卖比例）

---

#### 【三、仓位管理分析】
1. 单股仓位占比（最高单股占总仓位%）
2. 持股集中度（TOP3股票占总仓位%）
3. 止损执行率（设置止损且执行的比例）
4. 满仓操作天数占比

---

#### 【四、盈亏模式分析】
1. 盈利交易的共同特征
   - 平均盈利金额
   - 平均持股周期
   - 买入时机特征
2. 亏损交易的共同特征
   - 平均亏损金额
   - 平均持股周期
   - 卖出时机特征
3. 盈亏比（平均盈利 / 平均亏损）

---

#### 【五、行为偏差诊断】（核心！）

逐项诊断以下行为偏差，每项给出「有/无」判断 + 具体证据：

1. **过度交易**
   - 判断：
   - 证据：月均交易XX次，日内交易占比XX%

2. **损失厌恶**（亏了死拿）
   - 判断：
   - 证据：盈利持仓平均X天，亏损持仓平均X天

3. **羊群效应**（跟买热点）
   - 判断：
   - 证据：买入时市场情绪（涨停板跟风比例）

4. **锚定效应**（成本价执念）
   - 判断：
   - 证据：是否经常持有已亏损股票等待回本

5. **近期偏差**（近因效应）
   - 判断：
   - 证据：是否近期赚钱则模仿，近期亏钱则改变策略

---

#### 【六、综合评分】

| 维度 | 得分（0-100） | 评价 |
|------|--------------|------|
| 交易频率控制 | XX | 合理/过度 |
| 仓位管理 | XX | 好/一般/差 |
| 止损执行 | XX | 好/一般/差 |
| 买卖时机 | XX | 好/一般/差 |
| 情绪控制 | XX | 好/一般/差 |

**综合评分：XX分**（满分100）

---

#### 【七、核心问题总结】（Top 3，必须量化）

1. 您最突出的行为问题是：________
   - 量化证据：________
   - 亏损贡献度：约___%

2. 第二突出的行为问题是：________
   - 量化证据：________

3. 第三突出的行为问题是：________
   - 量化证据：________

---

#### 【八、改进建议】

根据上述分析，给出3-5条具体、可执行的改进建议：

**建议1：________**
- 具体做法：________
- 预期效果：________

**建议2：________**
...

---

#### 【九、下次分析重点关注】

下次分析时，请特别关注：
1. 止损执行率是否有提升
2. 平均持股周期是否变化
3. 仓位集中度是否改善

### 2.4 Few-Shot示例

每次调用AI时，必须在Prompt中附上一个示例报告，让AI按照此格式输出：

```
【示例报告片段】

## 五、行为偏差诊断

1. **过度交易** - 有
   证据：月均交易18次，其中日内交易占比60%。A 股民平均月交易4-6次，
   您的交易频率是平均水平的3倍。每次交易都在消耗手续费（万三~千一），
   按18次/月计算，年手续费消耗约账户总值的3-5%。

2. **损失厌恶** - 有
   证据：盈利持仓平均持有5天，亏损持仓平均持有28天。
   典型的"赚点就跑、亏了就等"，导致止损不及时，盈利被截断。
```

## 三、Prompt调用规范

### 3.1 每次分析的标准调用格式

```
【系统提示词】
[见2.1节]

【用户交易数据】
[JSON格式数据]

【分析要求】
请严格按照报告框架（2.3节）生成分析报告，
重点关注行为偏差诊断（第五节），给出量化证据。
参考示例格式（2.4节）。

【输出语言】
全程使用简体中文，数字使用阿拉伯数字。
```

### 3.2 输出质量检查清单

生成报告后，AI必须自检：
- [ ] 是否覆盖了所有8个章节
- [ ] 每个维度是否有量化证据（不是空泛描述）
- [ ] 行为偏差判断是否有具体数字支撑
- [ ] 建议是否具体可执行（不是"注意风险"这种废话）
- [ ] 是否有不该出现的内容（荐股、价格预测等）

## 四、数据处理规范

### 4.1 重复数据去重

当用户上传多张截图时，按以下规则去重：
- 同一股票 + 同一日期 + 同一价格 → 保留一条
- 去重前记录数 / 去重后记录数 = 去重率（应 < 15%）
- 超过15%提示用户可能有重复截图

### 4.2 数据校验

用户确认界面显示AI识别的交易列表，用户可：
- 修改任意一条的数据
- 删除错误条目
- 添加遗漏条目

**只有用户点击"确认"后，数据才进入分析流程。**
