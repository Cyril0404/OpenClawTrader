## 一、核心问题

### 1.1 现有代码方向错误

| 现有文件 | 当前功能 | 开发手册要求 | 问题 |
|---------|---------|------------|------|
| TechnicalAnalysisView | K线图技术指标 | 无（不做技术分析） | ❌ 方向错 |
| SentimentAnalysisView | 情绪分析 | 无（不做情绪分析） | ❌ 方向错 |
| TradingAnalysis.swift | 风格+风险评估 | 行为偏差诊断 | ⚠️ 部分对 |
| TradingService | 交易服务 | AI报告生成 | ❌ 缺失 |

### 1.2 产品红线违反

现有代码存在**荐股思路**：

```swift
// TradingAnalysis.swift 里的 TradingSuggestion
title: "芯源微今日涨幅4.63%，注意获利了结"  // ❌ 这是荐股
description: "建议设置短期止盈位在180元附近"  // ❌ 具体标的价格
```

**开发手册明确规定：**
- ❌ 不能荐股
- ❌ 不能给具体买卖价格
- ✅ 只能分析行为偏差
- ✅ 只能说"你为什么会亏"

## 四、核心模块设计

### 4.1 行为偏差诊断（核心！）

开发手册要求的5项偏差：

```swift
struct BehaviorBiasReport: Codable {
    // 1. 过度交易
    var overTrading: BiasResult
    
    // 2. 损失厌恶
    var lossAversion: BiasResult
    
    // 3. 羊群效应
    var herdBehavior: BiasResult
    
    // 4. 锚定效应
    var anchoringEffect: BiasResult
    
    // 5. 近期偏差
    var recencyBias: BiasResult
}

struct BiasResult: Codable {
    var hasBias: Bool           // 有/无偏差
    var score: Double           // 0-10分
    var evidence: [String]      // 量化证据
    var description: String     // 通俗解释
}
```

## 六、资金账号验证

### 6.1 验证规则

```swift
struct AccountVerificationService {
    // 格式：20660 + 9位数字
    static let pattern = "^20660\\d{9}$"
    
    static func verify(_ account: String) -> Bool {
        return account.matches(regex: pattern)
    }
}
```

### 6.2 验证流程

```
用户输入资金账号
    ↓
前端正则验证
    ↓
格式正确 → 解锁真实功能
格式错误 → 提示"账号格式不正确"
```
