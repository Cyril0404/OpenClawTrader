## 二、今日开发总结 (2026-03-30)

### 2.1 iOS WebSocketChatService 修复

**文件**: `iOS/Services/WebSocketChatService.swift`

**问题**: 聊天消息发送成功，但 AI 回复始终显示 "(空响应)"

**修复内容**:

1. **添加 `hasStreamedContent` 标志** - 追踪是否已有流式内容到达
2. **chat 事件处理改进**:
   - 同时调用 `streamCallback`（实时更新 UI）和添加到 `incomingMessages`（让 `onReceive` 收到）
   - 更新 `lastAIResponse`
   - 当 `state == "final"` 时，如果已有流式内容，不调用空回调覆盖
3. **agent 事件处理**: 同样添加 `hasStreamedContent = true` 和 `lastAIResponse` 更新
4. **res 响应处理**: 如果已有流式内容且结果为空，忽略空结果
5. **发送消息时重置状态**: `lastAIResponse = nil`, `hasStreamedContent = false`
6. **添加调试日志**: 在 `handleGatewayMessage` 和 `chat` 事件处理中添加了详细的 print 日志

### 2.2 SimpleChatView 消息持久化修复

**文件**: `iOS/Features/Chat/SimpleChatView.swift`

**问题**: 切换页面后聊天记录被清空

**修复内容**:
- 将 `messages` 从局部 `@State` 移到共享的 `ChatInputState.messages`
- 所有 `messages` 引用改为 `chatState.messages`

### 2.3 调试过程

通过在 iOS 端添加调试日志，发现：

1. `chat.send` 成功发出：`{"type":"res","id":"xxx","ok":true,"payload":{"runId":"xxx","status":"started"}}`
2. 收到了 `chat` 事件，但 payload 内容为：
```json
{
  "runId": "xxx",
  "sessionKey": "agent:main:main",
  "seq": 1,
  "state": "final"
}
```
**关键发现**: 没有 `delta` 和 `text` 字段！只有 `state: "final"`，说明 Gateway 只是确认消息结束，但没有发送实际的响应内容。

### 2.4 Gateway Token 问题（未解决）

Mac mini 上运行 gateway-bridge 时发现：

```
[GATEWAY] Connect failed: {"code":"INVALID_REQUEST","message":"unauthorized: gateway token missing (provide gateway auth token)"}
```

**原因**: `~/.openclaw/openclaw.json` 中的 `gateway.auth.token` 对 Gateway 来说无效或已过期。

**尝试修复**:
1. 运行 `openclaw configure` - 失败，因为 `memos-local-openclaw-plugin` 插件有 Node.js 版本不匹配问题
2. 尝试 `npm rebuild` 修复插件 - 失败（Puppeteer Chrome 下载问题）

## 四、已知问题

### 4.1 Gateway Token 无效 ❌（阻塞）

**问题**: gateway-bridge 无法连接到 OpenClaw Gateway，报 "gateway token missing" 错误

**影响**: iOS App 发送的消息无法被 AI 处理，始终返回 "(空响应)"

**解决方案**:
1. 禁用 `memos-local-openclaw-plugin` 插件（Node.js 版本不匹配）
2. 运行 `openclaw configure` 重新获取有效的 gateway token
3. 用新 token 重启 gateway-bridge

### 4.2 API 端点不完整 ⚠️

**问题描述**：OpenClaw Gateway WebSocket 协议不支持以下方法：
- `/api/v1/workspaces` - 不支持
- `/api/v1/agents` - 不支持
- `/api/v1/models` - 不支持（通过 WebSocket）
- `/api/v1/workflows` - 不支持

**原因**：OpenClaw Gateway 的 WebSocket relay 协议只实现了部分方法：
- `status` ✅
- `chat` ✅
- `models` (OpenAI-compatible HTTP 端点存在，但需要直接 HTTP 访问 Gateway)

**影响**：iOS App 首页的 Workspaces/Agents/Models/Workflows 列表无法显示完整数据

**解决方案**：

**方案A（推荐）- 改 iOS App 使用 /v1/status**
```swift
// APIClient.swift 改造
// getWorkspaces() → 返回虚拟单 workspace
// getAgents() → 从 status.heartbeat.agents 解析
// getModels() → 从 status 获取默认模型
// getWorkflows() → 返回空数组，标注"暂不支持"
```

**方案B - gateway-bridge 加 HTTP 代理**
```
iOS → relay-server → gateway-bridge → Gateway HTTP 端口
```
需要 gateway-bridge 把 iOS 的 HTTP 请求代理到 Gateway 的 OpenAI-compatible 端点。

### 4.3 gateway-bridge 连接参数

**问题**：gateway-bridge 启动时需要指定参数，不支持配置文件

**当前启动命令**：
```bash
node gateway-bridge.js \
  --relay ws://150.158.119.114:3001 \
  --url http://127.0.0.1:18789 \
  --token <GATEWAY_TOKEN> \
  --gateway-id mac-mini-gw-2244
```
